# Cryptobot TFM

Sistema de trading algorítmico para criptomonedas desarrollado como Trabajo Fin de Máster en Inteligencia Artificial y Big Data.

El proyecto tiene como objetivo diseñar, desarrollar y evaluar un sistema completo de trading automático orientado a estrategias de scalping sobre el mercado spot de criptomonedas, utilizando datos históricos reales obtenidos desde la API de Binance.

El sistema integra distintas fases del pipeline de ciencia de datos y machine learning:

- Adquisición y almacenamiento de datos financieros
- Feature engineering basado en indicadores técnicos
- Análisis exploratorio de datos (EDA)
- Entrenamiento y evaluación de modelos supervisados
- Backtesting de estrategias
- Aprendizaje por refuerzo para la toma de decisiones
- Ejecución paper trading
- Dashboard de monitorización

El proyecto se desarrolla principalmente en Python y sigue una estructura modular orientada a reproducibilidad y escalabilidad.

# Setup and usage

## Python requirement

Python 3.11 is recommended for this project.

The project was developed in a local Python/Jupyter environment. Other recent Python versions may work, but Python 3.11 is the recommended option for reproducing the notebooks and the dashboard with fewer dependency issues.

Check your Python version with:

```bash
python --version
```

## Create and activate a virtual environment

From the repository root:

```bash
python -m venv .venv
```

On Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

On Windows CMD:

```cmd
.venv\Scripts\activate.bat
```

On Linux/macOS:

```bash
source .venv/bin/activate
```

## Install dependencies

With the virtual environment active:

```bash
python -m pip install --upgrade pip
pip install -r requirements.txt
```

Optional, if you want to make the environment available as a Jupyter kernel:

```bash
python -m ipykernel install --user --name cryptobot-tfm --display-name "Python (cryptobot-tfm)"
```

## Environment variables

The project uses a local `.env` file for Binance Spot Testnet credentials and paper trading configuration.

A safe template is provided in:

```text
.env.example
```

Create your local `.env` file from the template:

On Windows CMD:

```cmd
copy .env.example .env
```

On Windows PowerShell:

```powershell
Copy-Item .env.example .env
```

On Linux/macOS:

```bash
cp .env.example .env
```

Then edit `.env` and fill only the required Binance Spot Testnet credentials:

```env
BINANCE_TESTNET_API_KEY=
BINANCE_TESTNET_API_SECRET=
```

Important security warnings:

- Never commit the real `.env` file.
- Never upload API keys, secrets or credentials to GitHub.
- Use Binance Spot Testnet keys only.
- Do not use production Binance API keys in this academic project.
- Keep `ALLOW_LIVE_TESTNET_ORDERS=false` unless you deliberately want to enable Spot Testnet order validation.
- The project is intended to run in paper trading mode, without real capital.

The offline notebooks can be reproduced mostly from public historical data and generated local files. The `.env` file is mainly required for the testnet/paper execution layer and the live paper trader.

## Notebook execution order

The notebooks are numbered in execution order. To reproduce the full experimental pipeline, run them from `01` to `11`:

1. `notebooks/01_data_acquisition_doge_binance.ipynb`
2. `notebooks/02_feature_engineering.ipynb`
3. `notebooks/03_eda.ipynb`
4. `notebooks/04_baseline_ml.ipynb`
5. `notebooks/05_backtesting_ml_v1.ipynb`
6. `notebooks/06_backtesting_ml_v2.ipynb`
7. `notebooks/07_backtesting_ml_v3.ipynb`
8. `notebooks/08_baseline_RL_v1.ipynb`
9. `notebooks/09_baseline_RL_v2.ipynb`
10. `notebooks/10_baseline_RL_v3.ipynb`
11. `notebooks/11_testnet_paper_trading.ipynb`

Several notebooks depend on files generated by previous notebooks, especially datasets under `data/`, trained models under `models/` and result artifacts under `results/`. If something is missing, run the previous numbered notebook first.

## Running the live paper trader

The live paper trader can be launched from Windows using:

```cmd
launch_live_paper_trader.bat
```

This launcher starts the live paper trading process, which reads market data, generates live features, loads a selected model/policy configuration, simulates trades through the paper broker and writes execution events to JSONL logs.

Equivalent command from the repository root:

```bash
python scripts/run_live_paper_trader.py
```

The live paper trader does not need the dashboard to be open. It writes logs independently, and the dashboard can read those logs afterwards.

## Running the dashboard

The Streamlit dashboard can be launched from Windows using:

```cmd
launch_dashboard.bat
```

Equivalent command from the repository root:

```bash
streamlit run app/streamlit_dashboard.py
```

The dashboard reads generated logs, shows market data, visualizes paper trades, displays equity evolution and allows inspection of the deployed paper bot state.

Typical live demo order:

1. Start the live paper trader with `launch_live_paper_trader.bat`.
2. Start the dashboard with `launch_dashboard.bat`.
3. Select the current `run_id` in the dashboard to monitor the bot execution.

# Datasets

Current datasets are based on DOGEUSDT 5-minute candles obtained through Binance API.

Examples:

- `DOGEUSDT_5m_binance_2017_2026_features.csv`

The raw and processed datasets can be regenerated by running the numbered notebooks in order. Large generated files may be excluded from Git depending on the repository `.gitignore` configuration.
